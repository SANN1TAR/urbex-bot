# User Spec — Полная переработка Urbex Bot

## Что делаем

Полная переработка бота с нуля: исправление всех критических багов, переезд на надёжную базу данных, улучшение качества данных (без выдуманных координат), упрощение функционала.

## Зачем

Бот в текущем состоянии нестабилен: регулярно падает, выдаёт фейковые координаты и адреса, показывает одинаковые объекты под разными названиями. Пользоваться невозможно. Нужна переработка под корень.

## Как должно работать

### Регистрация
- Новый пользователь вводит /start → соглашается с дисклеймером → вводит город
- Поддерживаемые города: Москва, Московская область (для начала)
- Алиасы: мск/москва/moscow → Москва; мо/подмосковье/московская область → Московская область
- Повторный /start → приветствие без повторной регистрации

### Главный экран
Три кнопки:
- 🏚️ Заброшка
- 🏙️ Сменить город
- (убрана кнопка "🔍 Поиск по названию")

### Тиндер-интерфейс
- Нажал "Заброшка" → показывает один объект
- Кнопки под объектом: "➡️ Следующий" и "🔄 Заново"
- Следующий → следующий объект из кеша
- Заново → сбросить показанные, начать с новых объектов

### Карточка объекта
Что показываем:
```
[фото, если есть]

Название объекта

🗺 55.1234, 37.5678   ← координаты (приоритет)
  ИЛИ
📍 ул. Примерная, 1   ← адрес (если нет координат)

⚡ Инфа может быть устаревшей. Перед вылазкой проверяй.
```

Что убрано: описание, информация об охране.

**Правило показа объекта:** показывать ТОЛЬКО если есть хотя бы одно из:
- Координаты (реальные, с urban3p.ru — не выдуманные)
- Адрес (с страницы urban3p.ru)

Объекты без локации — не показывать и не сохранять в базу.

### Архив и обновление данных

Бот ведёт постоянно растущий архив объектов. База только пополняется — объекты не удаляются.

- **Архив**: все найденные объекты хранятся в Supabase бессрочно
- **Пополнение**: раз в 7 дней бот ищет новые объекты для города и добавляет в архив (старые не трогает)
- **При нажатии "Заброшка"**: если последнее пополнение было < 7 дней назад → берём из архива; если > 7 дней → ищем новое и добавляем
- **Если urban3p.ru недоступен**: показываем что есть в архиве, с пометкой что данные могут быть устаревшими
- **Если архив пуст И сайт недоступен** → "Сервис временно недоступен, попробуй позже"
- **/restart** → сбрасывает историю показанных для пользователя, НЕ очищает архив; принудительно ищет новые объекты и добавляет в архив

### Дедупликация
- Один и тот же объект НЕ показывается дважды под разными названиями
- Если два объекта ближе ~100 метров друг к другу → считаем одним, показываем первый найденный

## Критерии приёмки

- [ ] Бот запускается без ошибок при наличии всех env vars
- [ ] При отсутствии любого env var → понятное сообщение об ошибке при запуске (не cryptic crash)
- [ ] Нажатие "Заброшка" → показывает объект с координатами ИЛИ адресом (не пустую карточку)
- [ ] НИ ОДИН показанный объект не имеет выдуманных/случайных координат
- [ ] Один и тот же объект не появляется дважды за одну сессию просмотра
- [ ] "Следующий" работает без ошибок при быстром нажатии
- [ ] Данные пользователей сохраняются после деплоя (Supabase, не SQLite)
- [ ] Объекты в базе сохраняются после деплоя
- [ ] Кнопка "🔍 Поиск по названию" отсутствует
- [ ] /help показывает актуальную справку без упоминания поиска по названию
- [ ] Бот не падает при недоступности urban3p.ru — показывает что есть в базе

## Ограничения

- Бюджет: $0 — только бесплатные сервисы (Supabase free, Tavily free tier, Railway free/hobby)
- Нет поиска по названию (возможно вернём в будущем)
- Нет Nominatim (убираем как источник выдуманных координат)
- Нет Groq / LLM (убираем полностью вместе с поиском по названию)
- Только реальные координаты с urban3p.ru /onmap

## Риски

- urban3p.ru может заблокировать Railway IP (как Overpass в v0.4) → нужен fallback на кеш
- Tavily free tier: 1000 запросов/месяц — достаточно для одного пользователя, мало для 100+
- Supabase free: 500MB, 50k rows — хватит на долго

## Технические решения (для обсуждения)

- База данных: SQLite → Supabase PostgreSQL (asyncpg)
- Убрать: groq, поиск по названию, Nominatim, _tavily_photo()
- Добавить: координатная дедупликация (Haversine), lat/lon как отдельные колонки

## Как проверить (агент)

1. Запустить бота локально с .env
2. Написать /start, зарегистрироваться с городом "Москва"
3. Нажать "Заброшка" → объект должен появиться с координатами или адресом
4. Нажать "Следующий" 5 раз → не должно быть повторов
5. Нажать "Следующий" быстро два раза → не должно быть краша
6. Остановить Railway-деплой и перезапустить → пользователь должен сохраниться в базе

## Как проверить (пользователь)

1. Открыть @TZRCTfinderBot
2. Нажать "Заброшка" — появился объект? Есть координаты или адрес?
3. Несколько раз нажать "Следующий" — нет повторов?
4. Скопировать координаты, открыть в Яндекс.Картах — место реальное?
