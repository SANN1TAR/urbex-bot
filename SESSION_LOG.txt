SESSION_LOG — Urbex Bot
=======================

[2026-04-30] Сессия 1-2 — Полная разработка
---------------------------------------------
СДЕЛАНО:
- Создан Telegram-бот @TZRCTfinderBot
- Стек: Python + aiogram 3 + Groq (llama-3.1-8b-instant) + Tavily + SQLite
- Деплой на Railway (всегда онлайн)

ФУНКЦИОНАЛ:
- Регистрация по городу (без имени). Поддержка аббревиатур: МСК, ЕКБ, СПБ и т.д.
- Интерфейс как тиндер — по одному объекту, кнопки "Следующий" / "Заново"
- Плашки: 🏚️ Заброшка, 🏗️ Крыша, 🔍 Поиск по названию, 🏙️ Сменить город
- /restart — сбрасывает память и начинает с нуля
- Дисклеймер при первом запуске (ответственность)
- Напоминание про VPN после регистрации

ПОИСК:
- Иерархия источников: Викимапия → Урбантрип → Telegram → VK → общий поиск
- Groq обрабатывает результаты, отвечает только на русском
- Фото через Tavily images
- Координаты только из источника (не выдумывает)
- Адрес: только конкретная улица/район, иначе "место не помечено на карте"
- Python-фильтр запрещённых объектов: Москва-Сити, ВДНХ, МГУ, Останкино и тд
- Глобальная память показанных (до рестарта не повторяет)

КЛЮЧИ (в Railway Variables):
- TELEGRAM_TOKEN
- GROQ_API_KEY
- TAVILY_API_KEY

ЧТО ДАЛЬШЕ (возможные улучшения):
- Качество фото — иногда не те
- Координаты — Groq иногда галлюцинирует
- Качество объектов — зависит от источников
