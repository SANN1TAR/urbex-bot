[2026-05-01] Сессия 3 — Большой рефакторинг
--------------------------------------------
СДЕЛАНО:
- Удалён функционал крыш/руфинга полностью
- Groq заменён на Gemini Flash → не заработал (quota 0) → вернули Groq
- Модель сменена с llama-3.1-8b на llama-3.3-70b → исчерпал лимиты за час
- Модель сменена на gemma2-9b-it → задеплоена Groq, не работает
- Итог: вернулись на llama-3.1-8b-instant (500k/day, стабильная)

АРХИТЕКТУРА ПОИСКА:
- Было: 7 вызовов LLM за один поиск (по одному на каждый источник)
- Стало: 1 вызов LLM на все источники сразу → экономия токенов в 7 раз
- Промпт ужесточён: запрет выдумывать координаты и адреса
- Расширен BANNED_WORDS: деревни, снос, кинотеатры, реновация и тд
- Улучшена дедупликация: нормализация имён + дедупликация по адресу

ПРОБЛЕМЫ РЕШЁННЫЕ:
- Бот выдавал "Попробуй позже" из-за rate limit Groq → архитектура 1 вызов
- Дублирование объектов под разными именами → нормализация названий
- Выдуманные координаты → запрет в промпте + валидация
- Объекты из других городов (Зеленоград вместо Москвы) → запрет в промпте
- Мусорные описания "в неизвестное время" → _clean_description()
- "Место не помечено на карте" — убрана раздражающая фраза

[2026-05-01] Сессия 4 — Переход на OSM → провал → urban3p.ru
-------------------------------------------------------------
ПОПЫТКА OSM/OVERPASS:
- Переделан поиск на OpenStreetMap (Overpass API + Nominatim)
- Overpass API не работает с Railway: 406/400/403/таймаут на всех серверах
- Причина: Railway IP блокируется Overpass серверами
- Попробовано 5+ серверов Overpass — всё безуспешно

ПЕРЕХОД НА БАЗУ ДАННЫХ:
- Добавлена таблица objects в SQLite для кеша заброшек
- Кеш TTL = 7 дней, фоновое обновление раз в сутки
- При первом запросе по городу → собирает и сохраняет → дальше из базы

ИТОГОВОЕ РЕШЕНИЕ — urban3p.ru через Tavily:
- Tavily ищет объекты на urban3p.ru (крупнейшая база заброшек РФ)
- Парсинг страниц объекта напрямую через httpx: OG фото + /onmap координаты
- Nominatim как запасной вариант для координат
- Фильтры: только /objectXXXX URL, без статей/топов/дзена
- Убраны описания из карточки — только название + локация + фото
- Добавлена команда /help с объяснением для пользователей

ТЕКУЩИЙ СТЕК:
- Поиск: Tavily (urban3p.ru) + httpx парсинг страниц
- Координаты: /onmap парсинг + Nominatim fallback
- Фото: OG:image парсинг + Tavily images fallback
- LLM (Groq llama-3.1-8b): только для команды /поиск по названию
- Кеш: SQLite с TTL 7 дней

НЕРЕШЁННЫЕ ПРОБЛЕМЫ:
- Координаты есть не у всех объектов (urban3p.ru прячет их за /onmap POST)
- Часть объектов устарела (снесены, восстановлены) — данные из urban3p.ru
- Nominatim плохо находит промышленные заброшки по названию

ЧТО МОЖНО УЛУЧШИТЬ:
- Фильтр по году публикации (отсеивать объекты старше 2020)
- Добавить другие источники: urban3p + дополнительные сайты
