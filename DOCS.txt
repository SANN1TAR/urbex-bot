URBEX-BOT — Документация v0.5
==============================

СТЕК:
-----
Python + aiogram 3 + Tavily + Groq + SQLite
Деплой: Railway (всегда онлайн)
Бот: @TZRCTfinderBot

ФАЙЛЫ:
------

bot.py
  Главный файл. Запускает бота, обрабатывает команды и кнопки.
  Зависит от: database.py, search.py
  Запуск: python bot.py

database.py
  SQLite база. Две таблицы:
    users — telegram_id, город
    objects — кеш найденных заброшек (city, name, coords, address, image и тд)
  Функции: init_db, get_user, save_user, get_objects, save_objects,
           get_city_count, get_cache_age_days, clear_city_cache,
           get_all_cached_cities

search.py
  Поиск объектов через Tavily (urban3p.ru) + парсинг страниц + SQLite кеш.
  Функции:
    search_objects(obj_type, city, shown) — берёт из базы или собирает заново
    search_by_name(name, city) — поиск конкретного объекта (Tavily + Groq)
    _fetch_and_cache(city) — собирает объекты с urban3p.ru и сохраняет в базу
    _scrape_object_page(url) — парсит страницу объекта: OG фото + /onmap координаты
    _get_coords_nominatim(name, city) — координаты через Nominatim если /onmap не дал

requirements.txt
  aiogram, groq, tavily-python, httpx, aiosqlite, python-dotenv

.env
  TELEGRAM_TOKEN, GROQ_API_KEY, TAVILY_API_KEY
  НЕ КОММИТИТЬ в git.

urbex_bot.db
  SQLite база. Создаётся автоматически. На Railway сбрасывается при деплое.

КНОПКИ БОТА:
------------
🏚️ Заброшка — ищет заброшенные объекты в городе пользователя
🔍 Поиск по названию — найти конкретный объект по названию
🏙️ Сменить город — изменить город поиска
➡️ Следующий — следующий объект
🔄 Заново — сбросить показанные, начать заново

КОМАНДЫ:
--------
/start   — начало, регистрация города
/restart — сброс кеша и памяти показанных объектов
/help    — справка для пользователя

ПЕРЕМЕННЫЕ RAILWAY:
-------------------
TELEGRAM_TOKEN — токен бота от BotFather
GROQ_API_KEY   — ключ Groq (модель llama-3.1-8b-instant, для поиска по названию)
TAVILY_API_KEY — ключ Tavily (основной поиск и фото)

КАК РАБОТАЕТ ПОИСК:
-------------------
1. Пользователь нажимает "Заброшка"
2. Бот проверяет SQLite кеш для города (TTL = 7 дней)
3. Если кеш пустой или устарел:
   а. Tavily ищет на urban3p.ru по запросу "заброс заброшенная {город}"
   б. Для каждого объекта парсится страница urban3p.ru: OG фото + /onmap координаты
   в. Если координат нет — Nominatim по названию
   г. Объекты сохраняются в SQLite
4. Из базы выдаются 3 случайных непоказанных объекта
5. Раз в 24 часа фоновая задача обновляет кеш для всех известных городов

ФИЛЬТРЫ:
--------
- Только страницы /objectXXXX на urban3p.ru
- Фильтр BANNED_WORDS (Москва-Сити, госучреждения, торговые центры и тд)
- Фильтр JUNK_RE (статьи-списки, топы, дзен, youtube)
- Проверка что объект относится к нужному городу
